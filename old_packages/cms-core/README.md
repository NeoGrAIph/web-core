# `@synestra/cms-core`

Базовая CMS‑часть, общая для всех сайтов:
- общие коллекции (минимум: `users`),
- общие access‑функции/хуки,
- общие типы/хелперы для Payload.

Почему отдельный пакет:
- `admin.user` коллекция нужна всем Payload apps;
- единые правила доступа, чтобы не расходились по приложениям;
- проще тестировать и развивать схему.

Откуда берём паттерны:
- Payload template `website` (Users/access patterns): `docs/research/templates/payload-website.md`.

Что нельзя:
- не добавлять сюда app‑специфичную бизнес‑логику (это живёт в конкретном app или отдельном доменном пакете).
