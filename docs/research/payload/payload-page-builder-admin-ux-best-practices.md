# Payload CMS 3: UX админки для Page Builder (Blocks/Array) — заметки для `web-core`

Цель: зафиксировать UX‑практики для редактора при большом количестве блоков/полей, чтобы страницы было удобно собирать и поддерживать.

## 1) Block идентичность

- `slug` блока — публичный контракт данных (`blockType`).
- Читаемые `labels.singular/plural` обязательны для UX “Add block”.

Источник: https://payloadcms.com/docs/fields/blocks

## 2) Удобство навигации по странице

Рекомендуемые приёмы:
- использовать `admin.group` у блоков для группировки списка;
- при необходимости включать `blockName` (как “лейбл строки” блока), но не делать это обязательным везде без причины;
- для `array`/сложных вложенных структур добавлять `RowLabel` (чтобы строки были различимы).

## 3) Канон `web-core` (как применяем)

- Минимум: `labels` + группировка через `admin.group` при росте каталога.
- `blockName` включаем точечно (длинные страницы, потребность в именовании секций, edge-cases с `defaultValue`).
- Нормативный документ структуры каталога/registry: `docs/architecture/payload-page-builder-catalog.md`.
