# Инструкция по эффективному использованию результатов исследований

В проекте поддерживается реестр исследований в docs/research/web-research. Используй его как первую точку входа для ориентации, поиска источников и планирования работ.

## Как работать с реестром

1) Начинай с общего реестра
   - Открой docs/research/web-research/README.md.
   - Найди нужный сайт в таблице Site Registry.
   - Ищи страницы по Section, Tags и Summary в таблице Page Registry.

2) Теги — главный фильтр
   - Теги быстрее всего помогают найти релевантные страницы.
   - Старайся использовать уже существующие теги.
   - Если подходящего тега нет — добавь новый в docs/research/web-research/tags.md.

3) Работай через локальный README сайта и файл страницы
   - Открой docs/research/web-research/<domain>/README.md и найди строку нужной страницы.
   - Перейди в файл страницы в docs/research/web-research/<domain>/pages/<slug>.md.
   - Используй Summary, Key Sections и Useful For для понимания релевантности.
   - Doc Path помогает искать соседние страницы в том же разделе.

4) Помни о цели
   - Реестр фиксирует полезные источники, а не готовые ответы.
   - Summary должен объяснять, о чём страница и чем она полезна.
   - Не копируй большие куски текста; используй краткий пересказ.

5) Поддерживай актуальность
   - При использовании страницы обновляй Last Reviewed.
   - Если страница изменилась — обнови Summary, Key Sections и Tags.

6) Расширяй реестр системно
   - Добавляй страницы, которые закрывают пробелы или нужны для задачи.
   - Для нового сайта создавай папку, pages/, и локальный README.
   - На каждую страницу создавай отдельный файл в pages/.
   - Обновляй локальный README (таблица) и общий реестр.

## Быстрые пути
- Общий реестр: docs/research/web-research/README.md
- Реестр тегов: docs/research/web-research/tags.md
- README сайта: docs/research/web-research/<domain>/README.md
- Файлы страниц: docs/research/web-research/<domain>/pages/<slug>.md
