# `@synestra/eslint-config`

Единое место для ESLint‑правил монорепозитория.

Зачем:
- одинаковые правила для всех `apps/*` и `packages/*`;
- меньше копирования `eslint.config.*` и меньше “расхождений” между проектами;
- централизованное изменение правил.

Статус:
- Сейчас пакет экспортирует базовый flat‑config для Next.js + TypeScript (паттерн взят из официальных Payload templates).
- Дальше добавим:
  - правила импорт‑границ (apps не импортируют другие apps),
  - форматирование (Prettier или Biome — решение отдельно),
  - более строгие правила для `packages/*` (не Next).

Важно:
- Конфиг не должен зависеть от секретов/окружения.
