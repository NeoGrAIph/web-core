# `packages/`

Здесь живут **общие пакеты** (shared code) монорепозитория.

Назначение:
- переиспользование кода между сайтами/приложениями (`apps/*`);
- снижение “coordination tax” и дублирования;
- единые контракты (UI, CMS‑схемы, утилиты, конфиги).

Правила:
- пакеты не должны импортировать код из `apps/*` (только наоборот);
- секреты не храним нигде в `web-core`;
- для Next.js приложений пакеты, как правило, используются **как исходники** (`main/types/exports` указывают на `src/*`) и транспилируются Next’ом.

Связанные материалы:
- `docs/research/research.md` (best practices монорепо и Turbo)
- `docs/research/templates/*` (что именно требует Payload: migrations, media storage, env vars)
