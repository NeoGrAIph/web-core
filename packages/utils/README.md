# `@synestra/utils`

Пакет с общими **чистыми** утилитами (без UI/DOM/Next/Payload зависимостей).

Зачем:
- переиспользование функций между apps (corporate/shop/saas/landings);
- тестируемость (unit tests для функций проще и быстрее);
- сокращение дублирования (например форматирование дат, slugify и т.д.).

Правила:
- не зависеть от Next/Payload напрямую;
- держать функции детерминированными, без “скрытых” глобальных зависимостей;
- не хранить секреты и не читать env vars внутри утилит.
